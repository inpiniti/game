>
> 게임명: **딸기 서바이벌** (영문: `ddalki-survival`)
>
> 작성일: 2026-07-08 · [v1](./README.md) 대체 · 설계 기준: [docs/design/supabase-quiz-fsd-v7.md](../design/supabase-quiz-fsd-v7.md)

>
> **v1 대비 변경점**
> 1. **직접입력 모드: 정답 변형 수만큼 입력창 N개** — `anywhere`(뜻 3개)면 입력창 3개. 순서 무관, 전부 맞아야 정답 (8절)

## 1. 공통 규칙

### 반응형 브레이크포인트

| 구간 | 폭 | 레이아웃 |
|---|---|---|
| 모바일 | ~767px | 1열, 하단 탭바 |
| 태블릿 | 768~1023px | 2열 그리드, 상단 헤더 |
| 데스크톱 | 1024px~ | 3열 그리드, 상단 헤더 |

### 네비게이션

- **데스크톱·태블릿**: 상단 헤더 (로고 / 문제집 / 랭킹 / 기록 / [관리: admin만] / 닉네임·로그아웃)
- **모바일**: 상단 간소 헤더(로고 + 닉네임) + **하단 탭바** `[홈] [문제집] [랭킹] [기록]`
  - 게임·복습 세션 중에는 탭바 숨김 (전체 화면 사용)
  - 관리 메뉴는 admin 계정일 때 로비에 관리 카드로 노출
- 로그인/회원가입 화면에는 헤더·탭바 없음

### 터치·게임 대응

- 터치 타깃 최소 44px, 게임 선택지 버튼은 56px 이상
- 서바이벌: 데스크톱 키보드(방향키) / **터치 기기는 가상 조이스틱** 표시
- Phaser `Scale.RESIZE` 유지 — 세로·가로 모두 플레이 가능
- iOS 노치·홈바 세이프에어리어 패딩
- 테마: 게임 톤에 맞춘 **다크 단일 테마**

### 화면 흐름 (v2)

```
로그인 ⇄ 회원가입
   ↓ 성공
 로비 ──→ [서바이벌 시작] ──→ 문제집 선택 ──→ 방식 선택 모달 ──→ 서바이벌 ──→ 결과
   │                              ↑              (게임 종류·방향·방식)  └→ 단어장 ──→ 결과
   │        (탭/헤더에서 직접) ────┘
   ├──→ 복습 (오늘의 복습 / 취약 단어) ──→ 복습 세션(단어장 UI) ──→ 결과
   ├──→ 랭킹 · 기록
   └──→ 관리 (admin만): 공식 문제집 목록 / 업로드
```

### 라우트 매핑

| 화면 | 라우트 | 가드 |
|---|---|---|
| 로그인 / 회원가입 | `/login` `/signup` | 비로그인만 |
| 로비 | `/` | RequireAuth |
| 문제집 선택 | `/sets` | RequireAuth |
| 서바이벌 / 단어장 | `/play/survival/:setId` `/play/vocab/:setId` | RequireAuth |
| 복습 | `/review` | RequireAuth |
| 결과 | `/result` (직전 세션 state) | RequireAuth |
| 기록 / 랭킹 | `/history` `/ranking` | RequireAuth |
| 관리 | `/admin/sets` `/admin/upload` | RequireAdmin |

## 6. 방식 선택 (모달 — v2: 게임 종류 선택 추가)

```
+------------------------------------+
|        게임 방식 선택          [X]  |
+------------------------------------+
|                                    |
|  게임   (•) ⚔ 서바이벌             |
|         ( ) 📖 단어장              |
|                                    |
|  방향   (•) 영 → 한   ( ) 한 → 영  |
|                                    |
|  방식   (•) 자동 (추천)             |
|         ( ) 선택지   [ 3 ▼ ]개     |
|         ( ) 직접입력                |
|         ( ) 랜덤                   |
|                                    |
|  [          게임 시작          ]   |
|                                    |
+------------------------------------+
```

표시 규칙:

| 상황 | 노출 항목 |
|---|---|
| 로비 [서바이벌 시작]으로 진입 | 게임 = 서바이벌 선택된 채 (변경 가능) |
| 탭/헤더에서 문제집으로 바로 진입 | 게임 기본값 = 마지막 선택 |
| 게임 = 서바이벌 | 방식 숨김 (3지선다 고정 안내), 방향만 노출 |
| 게임 = 단어장 | 방향 + 방식 모두 노출 |
| 일반 문제집(learn_lang=null) | 게임 = 서바이벌만, 방향·직접입력·랜덤 숨김 |

- 마지막 선택값 기억(localStorage).

## 8. 게임 — 단어장 `/play/vocab/:setId` (복습 `/review` 세션도 동일 UI)

### 선택지 모드

```
+------------------------------------+
| [× 종료]      7 / 20      점수 350 |
+------------------------------------+
|                                    |
|   +----------------------------+   |
|   |                            |   |
|   |         anywhere           |   |
|   |                            |   |
|   +----------------------------+   |
|                                    |
|  [          어딘가에           ]   |
|  [          오답 A             ]   |
|  [          오답 B             ]   |
|                                    |
+------------------------------------+
```

### 직접입력 모드 (v2 — 정답 변형 수만큼 입력창)

`anywhere` = `어딘가에;아무데도;어디든지` (변형 3개) → 입력창 3개:

```
+------------------------------------+
| [× 종료]      7 / 20      점수 350 |
+------------------------------------+
|                                    |
|   +----------------------------+   |
|   |         anywhere           |   |
|   |      ※ 뜻 3개를 입력        |   |
|   +----------------------------+   |
|                                    |
|   답 1  ______________________     |
|   답 2  ______________________     |
|   답 3  ______________________     |
|                                    |
|   [           제출            ]    |
|                                    |
+------------------------------------+
```

**입력·채점 규칙**

- 입력창 수 = 정답 변형 수 (변형 1개면 기존처럼 1칸).
- **순서 무관** — `아무데도`를 답 1에 써도 인정.
- 같은 답을 두 칸에 쓰면 1개만 인정 (정규화 후 중복 판정).
- **모든 칸이 정답 변형과 일치해야 정답(good)** — 하나라도 틀리거나 비어 있으면 오답(again).
- 키보드: Enter = 다음 칸 이동, 마지막 칸 Enter = 제출.

**채점 피드백 (칸별 ✓/✗ + 못 맞힌 변형 공개)**

```
+------------------------------------+
|   ❌ 아쉬워요                       |
|                                    |
|   답 1  어딘가에               ✓   |
|   답 2  어디에나               ✗   |
|   답 3  아무데도               ✓   |
|                                    |
|   못 맞힌 뜻:  어디든지             |
|   💬 You can sit anywhere.         |
|                                    |
|   [           다음 →           ]   |
+------------------------------------+
```

### 정답 공개 — 선택지 모드

```
+------------------------------------+
|   ✅ 정답!                          |
|                                    |
|   anywhere                         |
|   = 어딘가에 ; 아무데도 ; 어디든지   |  ← 변형 전체 공개
|                                    |
|   💬 You can sit anywhere.         |  ← 예문 (있으면)
|                                    |
|   [           다음 →           ]   |
+------------------------------------+
```

- 데스크톱: 카드 최대 폭 560px 중앙 정렬. 선택지 1/2/3 키, Enter 제출·다음.
- 모바일: 입력 모드에서 소프트 키보드가 올라와도 단어 카드가 보이도록 카드 상단 고정, 입력창 영역 스크롤.

### 복습 진입 화면 `/review` (세션 시작 전)

```
+------------------------------------+
| 헤더                                |
+------------------------------------+
| 📌 오늘의 복습 12장     [복습 시작]  |
+------------------------------------+
| ⚠ 취약 단어 (8회 이상 오답)          |
|  +------------------------------+  |
|  | anywhere 어딘가에;… (12회) 💬 |  |
|  | run      달리다;뛰다 (9회) 💬 |  |
|  | happy    행복한;기쁜 (8회) 💬 |  |
|  +------------------------------+  |
|  [    취약 단어만 재도전    ]       |
+------------------------------------+
```

- 취약 단어 목록은 "정독"용 — 단어·전체 뜻·예문을 그대로 보여준다 (출제 아님).
