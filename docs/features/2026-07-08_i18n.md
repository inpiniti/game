# 기능: 다국어 (i18n) — UI 언어를 유저의 언어로

> 작성일: 2026-07-08 · 기준 설계: [supabase-quiz-fsd-v9.md](../design/supabase-quiz-fsd-v9.md) (13절) · 분류: **대형** (횡단 관심사 — 전 화면의 문자열 전제가 바뀜)
>
> 첫 기능 델타 문서. 기준 문서(설계 v9·화면 v3)는 얼리고, 이 기능에서 바뀌는 것만 여기에 적는다.

---

## 1. 왜

지금 UI는 한국어 하드코딩이다. 일본 유저가 가입해도(country=JP) 화면은 전부 한국어로 나온다. 국가별 공식 문제집·국가 대항 랭킹까지 만들어 놓고 UI가 한국어뿐이면 해외 유저는 사실상 못 쓴다. **유저의 언어로 UI가 보이게 한다.**

## 2. 핵심 설계 결정 — 언어 3개념 분리

이 앱에는 서로 다른 "언어/국가" 개념이 3개 있다. 혼동하면 안 된다.

| 개념 | 저장 위치 | 용도 | 이번 기능에서 |
|---|---|---|---|
| **국가** (country) | `profiles.country` | 랭킹 집계, 공식 문제집 노출 | 변경 없음. **UI 언어의 기본값 힌트로만 사용** |
| **학습 언어** (learn_lang) | `quiz_sets.learn_lang` | 문제집이 가르치는 언어 | 변경 없음 |
| **UI 언어** (ui lang) | `localStorage` (신규) | 인터페이스 표시 언어 | **신규 개념** |

국가 ≠ UI 언어다 (한국에 사는 일본인, 미국 국적의 한국어 사용자). 국가는 기본값을 정할 때만 참고하고, 유저가 명시적으로 고른 언어가 항상 이긴다.

### 2-1. UI 언어 결정 정책 (우선순위)

```
1. 명시 선택        localStorage 'ddalki.ui-lang' — 프로필 시트에서 바꾼 값
2. 국가 매핑        profiles.country → 언어 (KR→ko, JP→ja, 그 외→en …) — 로그인 후
3. 브라우저 언어    navigator.language → 지원 언어 매핑 — 비로그인 화면(로그인/가입)
4. 폴백            en
```

- 로그인 전(로그인/가입 화면)은 2번이 불가하므로 3→4로 결정된다
- 로그인 직후 명시 선택이 없으면 country 매핑으로 자동 전환 (유저 요구: "국가에 따라 자기 언어")
- **스키마 무변경**: `profiles.ui_lang` 컬럼은 만들지 않는다(1차). 기기 간 동기화가 필요해지면 후속으로 `docs/sql/04-add-ui-lang.sql` 추가

### 2-2. 지원 언어 — 30종 (국가 목록 51개국 전체 커버) *(2026-07-09 확장 — 1차 3종에서 전 국가로)*

```
ko en ja zh-CN zh-TW de fr it es pt nl sv no da fi
pl cs el ru uk tr hi id th vi ms mn kk ar he
```

- 국가→언어 매핑은 `resolveLang.ts`의 `COUNTRY_TO_LANG`이 51개국 전부 담당. 다국어 국가는 최다 사용 언어 기준(BE→nl, CH→de, ZA/PH/SG→en, KZ→kk, IN→hi). 목록 밖 국가는 en
- **번들 보호 — 지연 로딩**: ko(원문·폴백)만 즉시 로드, 나머지 29종은 `import.meta.glob`으로 언어 전환 시 청크 로드 (`changeUiLang()` 경유 필수)
- **RTL(ar·he) 베스트에포트**: `changeUiLang`이 `document.dir`을 자동 전환. 정식 RTL 레이아웃 검수 전까지 베타
- 확장 = `locales/<lang>.json` 파일 추가 + SUPPORTED_LANGS/LANG_LABELS/매핑 한 줄. 그 외 코드 수정 없음
- 번역은 기계 생성(Claude) → 네이티브 검수 전까지 베타 취급. 복수형은 언어별 CLDR 접미사(`_one`/`_few`/`_many`/… — ru·uk·pl·cs 4종, ar 6종, he 4종)

### 2-3. 라이브러리·구조

- **i18next + react-i18next** (표준, `useTranslation()` 훅, 복수형 내장). 언어 감지는 내장 detector 대신 **커스텀 resolve** (2-1 정책이 country를 봐야 해서)
- 키 규칙: 화면/위젯 프리픽스 dot notation — `lobby.start`, `quiz.correct`, `pause.quit`. 언어당 단일 JSON
- FSD 배치: `src/shared/i18n/` — 모든 레이어가 소비하는 횡단 관심사라 shared

```
src/shared/i18n/
├─ index.ts          # i18next 초기화, React 바인딩
├─ resolveLang.ts    # 2-1 우선순위 구현 + 국가→언어 매핑
└─ locales/
    ├─ ko.json  en.json  ja.json
```

### 2-4. 게임 내 텍스트 — 사전 주입

`src/game/`은 Supabase처럼 **i18n도 모른다** (경계 유지). 게임 내 라벨(점수·웨이브·일시정지·게임오버)은 `createSurvivalGame(host, bridge)`의 bridge에 **번역된 문자열 사전을 주입**한다:

```ts
export interface SurvivalTexts { score: string; wave: string; paused: string; /* … */ }
// createSurvivalGame(host, bridge, texts)  ← PlayPage가 t()로 만들어 넘김
```

- 씬은 받은 텍스트를 그리기만 한다. 게임 도중 언어 변경은 미지원(다음 판부터 적용 — 플레이 중 언어를 바꿀 UI 진입 자체가 없음)
- 폰트: 게임 캔버스는 시스템 폰트 스택 사용 중이라 ko/en/ja 글리프 문제 없음

### 2-5. 날짜·숫자 포맷

기록/랭킹의 날짜·숫자는 `Intl.DateTimeFormat`/`Intl.NumberFormat`을 현재 UI 언어 locale로 생성한다.

## 3. 화면 변경 (screens-v3 델타)

레이아웃 변경 없음. 조각 하나만 추가:

```
[프로필 바텀시트]                      ← widgets/profile-sheet (기존)
  닉네임 변경  [_________] [저장]
  언어         [한국어 ▾]              ← ★ 신규 row: ko/en/ja select
  ─────────────────────────
  [로그아웃]
```

- 변경 즉시 전 화면 반영 (i18next `changeLanguage` → 리렌더). 새로고침·재로그인 후에도 유지 (localStorage)
- 언어 이름은 **그 언어 자신으로 표기** (한국어 / English / 日本語) — 어떤 언어 상태에서도 자기 언어를 찾을 수 있게

## 4. 플랜 — 단계 × 스킬 × 모델

> 모델 기준은 [build-plan-skills-models.md](../development/build-plan-skills-models.md)와 동일 (Opus=설계·계약, Sonnet=구현 주력, Haiku=기계적).
> 신규 스킬: **i18n-expert** (`daymade/claude-code-skills@i18n-expert`, 1.2K installs) — `.agents/skills/`에 설치됨. *(후보였던 `lobehub/lobe-chat@i18n`은 lobe-chat 저장소 전용이라 탈락)*

| 단계 | 작업 | 사용 스킬 | 에이전트 모델 | 완료 기준 |
|---|---|---|---|---|
| **i-0 기반** | i18next+react-i18next 셋업, `shared/i18n/` 구조, **resolve 정책(2-1) 구현**(커스텀 detector — localStorage→country→browser→en), 3개념 분리가 코드에 드러나게 명명 | i18n-expert, codebase-design | **Opus** | 언어 강제 지정 시 셸이 그 언어로 |
| **i-1 문자열 추출** | 전 페이지·위젯·가드의 하드코딩 문자열 → `t('키')` + `ko.json` (원문 그대로, **동작 무변경**) — 13페이지+위젯 수백 개, 누락 검출 grep 포함 | i18n-expert | Sonnet | ko 화면이 기존과 픽셀 동일 |
| **i-2 번역** | `en.json`, `ja.json` 생성 — 톤 매핑(해요체→friendly/です・ます), 게임 용어(축복·처치·웨이브) 용어집 먼저 고정 | i18n-expert, toss-design | Sonnet | 3언어 JSON 키 100% 균형 |
| **i-3 전환 UI** | 프로필 바텀시트 언어 select(자기 언어 표기) + 즉시 반영 + localStorage 저장 | toss-design, frontend-design | Sonnet | 시트에서 변경 → 전 화면 즉시 전환 |
| **i-4 게임 주입** | bridge에 `texts` 사전 추가(2-4 계약), SurvivalScene 라벨 치환 — **게임↔앱 계약 변경이므로 신중** | phaser-best-practices, codebase-design | **Opus** | ja로 플레이 시 HUD·일시정지·게임오버가 일본어 |
| **i-5 검증** | Intl 날짜·숫자 적용(기록·랭킹), en/ja 긴 문자열 레이아웃 깨짐 점검(버튼·탭·카드), 3언어 스모크, typecheck/build | verify, code-review | Sonnet | 3언어 전 화면 스모크 통과 |

- 순서 의존: i-0 → i-1 → (i-2 ∥ i-3) → i-4 → i-5. i-2와 i-3은 병렬 가능
- i-1이 물량 최대(위험도 낮음·정확성 중요) — 파일 단위로 쪼개 병렬 에이전트 가능

## 5. 하지 않는 것 (명시적 범위 제외)

- ❌ 문제집 콘텐츠 번역 — 유저 콘텐츠다. `learn_lang`/`country` 체계 그대로
- ❌ `profiles.ui_lang` 컬럼 (1차) — localStorage로 충분, 필요 시 후속
- ❌ URL 경로 언어 (`/ko/lobby`) — SEO 필요 없는 로그인 필수 앱
- ~~❌ RTL~~ → **베스트에포트로 격상** (2026-07-09): ar·he 추가에 따라 `document.dir` 자동 전환만 지원. 정식 RTL 레이아웃 검수(물리적 left/right CSS 정리)는 미착수
- ❌ 게임 도중 실시간 언어 전환 — 진입 경로 없음, 다음 판 적용
