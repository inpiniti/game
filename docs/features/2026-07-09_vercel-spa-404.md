# 기능: Vercel 새로고침 404 수정 — SPA 라우팅 폴백

> 작성일: 2026-07-09 · 분류: **소형** — 배포 설정 파일 1개 추가로 끝나는 수정

## 왜

react-router 기반 SPA라 `/history` 같은 경로는 브라우저가 아니라 JS가 처리한다. 로컬 dev 서버(Vite)는 SPA 폴백이 내장이라 새로고침해도 `index.html`을 주지만, Vercel은 `/history`라는 **실제 파일**을 찾다가 404를 낸다. 루트(`/`)만 `index.html`이 실존해서 살아남는 것.

## 작업

| 단계 | 작업 | 사용 스킬 | 에이전트 모델 (effort) | 완료 기준 |
|---|---|---|---|---|
| 1 | 루트에 `vercel.json` 추가 — 모든 경로를 `index.html`로 rewrite (정적 자산은 파일시스템 우선이라 영향 없음) | — | Haiku (low, thinking X) | Vercel 재배포 후 `/history` 등에서 새로고침 시 200 |

## 완료 기준

- 배포된 사이트에서 루트가 아닌 페이지(기록/랭킹/문제집 등) 새로고침 시 404 없이 화면 유지
- `/assets/*.js` 등 정적 파일은 rewrite에 걸리지 않고 그대로 서빙 (Vercel은 실존 파일을 rewrite보다 우선)
